## Common Tasks (Simplified)

### Fix a Typo

1. Go to the page on GitHub
2. Click the pencil icon ‚úèÔ∏è
3. Fix the typo
4. Scroll down and click "Propose changes"
5. Click "Create pull request" (twice)

**That's it!** Someone will review and merge your fix.

### Add New Information

1. Find a similar existing page
2. Copy its structure
3. Replace the content with your new information
4. Submit as a pull request

Or use one of our templates!

### Create a New Page

1. Choose a template from `_templates/`
2. Copy it to the right folder in `_docs/`
3. Fill in your content
4. Update the page title and date at the top
5. Submit

## What Happens After You Submit?

1. **Automatic Check**: A robot checks formatting (don't worry if it finds issues!)
2. **Human Review**: A maintainer reviews your contribution
3. **Feedback** (maybe): They might suggest small changes
4. **Merge**: Your changes go live on the website!
5. **Celebration**: You've improved documentation! üéâ

## Frequently Asked Questions

### "What if I break something?"

You can't! GitHub keeps a full history, so nothing can be permanently broken. Worst case, we can undo changes.

### "What if my formatting is wrong?"

The automated linter will catch most issues. Maintainers will help fix anything that needs fixing. Don't let formatting stop you from contributing!

### "What if I don't know how to use Git/GitHub?"

Use GitHub's web editor - it handles Git for you! Or check out the VS Code option, which makes it even easier.

### "What if my grammar isn't perfect?"

Content matters more than perfect grammar. Editors will help polish the language if needed.

### "What if I'm not sure my contribution is good enough?"

Submit it anyway! The worst that happens is someone suggests improvements. Your effort is valued regardless.

### "Will someone judge my technical skills?"

Never. We appreciate all contributions equally. Everyone starts somewhere, and documentation contributions are valuable regardless of technical background.
